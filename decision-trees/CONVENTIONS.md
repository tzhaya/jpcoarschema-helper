# フローチャート共通規約

`decision-trees/` 配下の各プロパティ入力ガイドが共通で従う規約です。各ページはこのファイルを参照し、ここで定義した凡例・用語・方針を重複定義せずに使います。

---

## 1. ページ構成

各プロパティのページは次の順で構成します。

1. **見出し・対象**（対象プロパティ、典拠リファレンスへのリンク）
2. **まず入力するもの**（代表的な入力先の早見表）
3. **記号凡例**（→ 本ファイルへのリンクで代替可）
4. **入力フローチャート**（Mermaid `flowchart TD`）
5. **入力プロセス対応表**（判断・質問・回答・アクション・要素／属性・入力例）
6. **使い分けの目安**（迷いやすいケース）
7. **入力例**（XML）
8. **注記（入力ルール）**
9. **参考**（公式ページ・[要素別ルール](../reference/)・[DOI対照表](../reference/JPCOAR_JaLC_Crossref_requirements.md)）

---

## 2. 記号凡例（Mermaid）

| 記号 | 意味 |
|------|------|
| 楕円 `([ ])` | 開始 / 終了 |
| ひし形 `{ }` | 判断（Yes / No や種類の分岐） |
| 長方形 `[ ]` | 処理（入力・設定する内容） |
| 平行四辺形 `[/ /]` | 入力・情報源 |

---

## 3. 記入レベル・繰返回数の凡例（スキーマ層）

| 記号 | 意味 |
|------|------|
| M | 必須（Mandatory） |
| MA | 該当する場合は必須（Mandatory if Applicable） |
| R | 推奨（Recommended） |
| 1-N | 1回以上、繰返可 |
| 0-N | 0回以上、繰返可（必須以外） |
| 0-1 | 0〜1回、繰返不可（必須以外） |

---

## 4. DOI要件の用語（DOI登録層）

[JPCOAR/JaLC対照表](../reference/JPCOAR_JaLC_Crossref_requirements.md) の語を次の意味で用います。混同しないこと。

| 用語 | 意味 |
|------|------|
| **必須** | 必ず入力が必要 |
| **条件付必須** | 特定条件下で必須（例: 親要素がある場合） |
| **任意** | 入力は任意（0以上 または 0〜1） |
| **推奨** | 任意だが入力が推奨される |

> 注意: 対照表では項目ごとに「任意」と「推奨」を使い分けています（例: タイトルの `xml:lang` は JaLC で**任意**、作成者名の `xml:lang` は JaLC で**推奨**）。安易に言い換えないこと。

---

## 5. 3層の原則（最重要）

各要素・属性の要件は、必ず次の3層を区別して記述します。混同が誤りの主因です。

| 層 | 典拠 | 例 |
|----|------|----|
| **スキーマ層** | [公式説明ページ](https://schema.irdb.nii.ac.jp/ja/schema) / [要素別ルール](../reference/) | `dc:title` は M（必須・1-N）、`xml:lang` は MA |
| **DOI登録層** | [対照表](../reference/JPCOAR_JaLC_Crossref_requirements.md) | タイトル `xml:lang` は JaLC＝任意 / Crossref＝必須 |
| **本ガイド運用方針** | 本ファイル | `xml:lang` は入力漏れ防止のため原則付与 |

---

## 6. `xml:lang` の運用方針

- スキーマ上は「該当する場合は必須（MA）」。
- DOI登録要件は要素・登録先ごとに異なる（対照表に従う）。一般傾向として **JaLC は任意〜推奨、Crossref は必須**、Crossref の一部要素は **`xml:lang="en"` 必須**。
- **本ガイドの方針**: 入力漏れと多言語混在を防ぐため、**原則としてすべての言語依存値に付与**する。
- **1言語＝1要素**: 1つの要素に複数言語を並べない。言語が異なる場合は要素を分ける（並列表記は非推奨）。

### 言語コード

| 用途 | コード例 |
|------|---------|
| 日本語 / 英語 / 中国語(簡体) | `ja` / `en` / `zh-cn` |
| 片仮名ヨミ | `ja-Kana` |
| ローマ字ヨミ | `ja-Latn` |

- ヨミ（`ja-Kana` / `ja-Latn`）を入れる場合は、必ず本文言語（通常 `ja`）の値も併記する。

---

## 7. 用語

| 用語 | 定義 |
|------|------|
| 並列タイトル | 同一資源の代表タイトルが複数言語で示されたもの。`dc:title` を言語別に繰り返す |
| その他のタイトル | 本タイトル以外（副題・目次・奥付タイトル等）。`dcterms:alternative` |
| ヨミ | 読みの情報。`ja-Kana` / `ja-Latn` |

> 用語はプロパティ追加に応じて追記します。**公式に存在しない概念を独自に追加しない**こと（追加する場合は「運用上の区分」と明記）。
